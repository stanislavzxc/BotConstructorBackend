# Telegram Bot Constructor (Backend)

Универсальный бэкенд для чат-ботов на Flask. Архитектура проекта заточена под легкий перенос (экспорт) бота между серверами: вместо Redis используется JSON-конфигурация, что позволяет запустить проект буквально одной командой.

## 🚀 Особенности
- **No-Redis**: Вся логика и структура узлов хранятся в `data.json`.
- **Node-based**: Управление сценарием через систему узлов (nodes).
- **Легкий деплой**: Идеально подходит для работы на слабых серверах или в Docker-контейнерах.
- **Динамический контент**: Поддержка плейсхолдеров в текстах сообщений.

## 🛠 Архитектура проекта
*   `bot.py` — Точка входа и инициализация Flask/Telegram клиента.
*   `data.json` — Основной файл настроек сценария бота.
*   `service/` — Ядро системы:
    *   `nodes/` — Обработка перемещений пользователя по сценарию.
    *   `answers/` — Валидация и обработка пользовательского ввода.
    *   `replace_placeholders.py` — Логика подстановки данных в сообщения.
*   `db/` — Слой работы с БД для хранения состояния пользователей.
*   `utils/config.py` — Загрузка токенов и переменных окружения.

## 📦 Установка и запуск

### 1. Подготовка окружения
Убедитесь, что у вас установлен Python 3.10 или выше.

```bash
# Клонирование репозитория (если применимо)
# git clone <url> && cd <project_dir>

# Создание виртуального окружения
python3 -m venv venv

# Активация окружения
source venv/bin/activate
```

### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 3. Настройка
Создайте файл `.env` (или отредактируйте `utils/config.py`), указав ваш `TELEGRAM_TOKEN`. Все сценарии бота настраиваются в файле `data.json`.

### 4. Запуск
Для работы с портами ниже 1024 или доступа к системным ресурсам может потребоваться sudo:
```bash
sudo ./venv/bin/python bot.py
```
> **Важно:** При использовании venv запускайте интерпретатор напрямую из папки `venv/bin/python`, чтобы избежать конфликтов с системными пакетами.
